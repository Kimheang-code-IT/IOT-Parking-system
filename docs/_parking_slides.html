Tools We Use

  • Backend: Python, FastAPI, SQLAlchemy, SQLite
  • Frontend: Nuxt 4, Vue 3, Nuxt UI
  • IoT: Wokwi ESP32, Python lane scripts
  • Payment: ABA PayWay (mock QR for demo)
  • Editor: VS Code, Cursor

Project Idea

  • Smart parking with IoT entry & exit gates
  • Auto record license plate and parking time
  • Print ticket QR (invoice id) at entry
  • Calculate fee and pay with ABA / KHQR at exit
  • Staff dashboard shows live data from database

System Flow

1 Car enters → IoT scan plate
↓
2 API saves session + print ticket
↓
3 Car exits → scan QR + verify fee
↓
4 Customer pays → gate opens
↓
5 Dashboard updates parking & invoices

Entry Flow (IoT)

1 Camera / Wokwi detects plate + vehicle type
↓
2 Device calls POST /api/iot/entry-scan
↓
3 API creates session PK-* + invoice IN-*
↓
4 Printer shows ticket — QR = invoice id
↓
5 Entry gate opens — car enters parking

Exit & Payment Flow

1 Scan ticket QR + check plate at exit
↓
2 POST /api/iot/exit-verify → fee amount
↓
3 Customer pays — ABA QR on Payment page
↓
4 Bank webhook confirms → invoice Paid
↓
5 open-gate — session Completed

Project Structure

  • backend/ — FastAPI, database, IoT API
  • frontend/ — Nuxt dashboard (Home, Parkings, Payment, Invoices)
  • wokwi/ — entry-gate & exit-gate simulator
  • docs/ — system documentation
  • data/ — SQLite file iot_parking.db

How to Use

  1. Start API: backend → run_dev.ps1
  2. Start UI: frontend → pnpm dev
  3. Open localhost:3000 for dashboard
  4. Wokwi: set PC LAN IP in main.cpp, press entry/exit button
  5. Test: python -m scripts.test_integration

Summary

  • One system: IoT + API + Web connected
  • Fees and invoices are automatic
  • Works on one laptop (SQLite, no cloud needed for demo)
  • 20/20 integration tests passed
  • Ready to add real camera, printer, and gate later
